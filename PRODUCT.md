# Product

## Register

product

## Users

Sunpebble's maintainer uses this kit to create many small HarmonyOS apps quickly.
The user is building production apps, not prototypes, and needs every new app to
start from the same brand, component vocabulary, shell, and launch checklist.

## Product Purpose

HarmonyKit is the shared ArkUI foundation for Sunpebble HarmonyOS apps. It turns
the brand system into reusable code and a copyable tool-app shell so each app can
focus on one narrow job while sharing layout, controls, states, and packaging
discipline.

Success means a new tool app can be created from a template, receive a distinct
app profile, and ship without rebuilding buttons, cards, empty states, settings,
or home-card patterns from scratch.

## Brand Personality

Small, polished, quiet. The shared Sunpebble base is cream, ink, sun, pebble, and
night. App personality is added as a thin layer on top, never by forking the
component system.

## Anti-references

- No one-off app UI forks.
- No decorative dashboards, marketing heroes, or component galleries as product.
- No broad framework built for imagined future features.
- No per-app styling drift for buttons, cards, rows, empty states, or shells.

## Design Principles

1. One shell, many narrow tools.
2. App identity is data, not copied UI code.
3. Prefer local-first, single-purpose workflows.
4. Build reusable states before reusable screens.
5. Add only the component needed by at least two apps.

## Accessibility & Inclusion

Target accessible product UI: readable contrast, text labels beside color states,
standard ArkUI controls where possible, reduced-motion-friendly state changes,
and layouts that tolerate Chinese and English copy.
